🧱 三、你要自己做的「核心系統」

我幫你拆成 6 大系統（這才是重點）

1️⃣ Data Ingestion System（資料接入）

👉 你要寫：

Crypto Connector（Binance 等）
Stock Connector（Yahoo / API）
TW Connector（官方資料）
Macro Connector

👉 每個都要：

REST 抓歷史
WebSocket 抓即時
retry / backoff
rate limit 控制
2️⃣ Stream Processing（資料流引擎）

👉 你要做：

queue（Kafka 或自己簡化版）
pipeline：
raw data → clean → normalize → distribute

👉 功能：

去重
排序（時間）
tick → K線
修正異常值
3️⃣ Normalization Core（🔥最關鍵）

👉 這是整個系統的「大腦」

你要統一：

symbol
market
timeframe
price precision
timezone
🔥 Symbol Mapping（超級關鍵）

不同來源：

BTCUSDT (Binance)
BTC-USD (Coinbase)

👉 你要做：

統一 → BTC/USD
4️⃣ Storage System（資料儲存）

你要做三層：

🔴 Hot（即時）
Redis
最新 candle
🟡 Warm（查詢）
ClickHouse（最佳）
👉 查 K 線超快
🔵 Cold（歷史）
檔案 / S3
5️⃣ Aggregation API（對外服務）

👉 這是你 chart 用的

GET /kline
GET /symbols
GET /markets

👉 要支援：

lazy load
多市場
多 timeframe
6️⃣ Realtime Gateway

👉 WebSocket server

功能：

訂閱 symbol
推送最新 K 線
fan-out（多 client）
🔄 四、完整資料流（你要做到這樣）
📡 即時資料
Exchange

 → Processing

📚 歷史資料
